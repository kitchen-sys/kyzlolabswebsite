Platform Overview

Kyzlo is a production-grade system built on Hexagonal Architecture (Ports & Adapters). Pure domain logic stays isolated from external infrastructure—brokers, data feeds, databases swap in and out without touching core business rules.

Quant Finance Engine

Event-driven trading with real-time market data streaming, strategy execution, and automated order routing. Connects to Alpaca for execution, TimescaleDB for tick storage, and supports multiple concurrent strategies.

Sports Analytics Engine

The same battle-tested architecture applied to sports forecasting. Pluggable data adapters ingest game stats, player metrics, and historical outcomes. Model layer remains pure—no vendor lock-in.

Forecasting Services

Research RAG system with semantic document search, LLM-powered analysis, and structured query logging. Ingest PDFs, query with natural language, track prediction accuracy over time.

Hexagonal Architecture

Clean separation between what the system does and how it connects to the outside world. Domain logic has zero I/O dependencies. Swap brokers, databases, or data feeds without rewriting business rules.

Domain Layer

Pure business logic with no external dependencies. Models (Tick, Order, Position, Fill), Events (TickEvent, SignalEvent, FillEvent), Strategies (GoldenCross, MeanReversion), and Risk Rules (position limits, drawdown thresholds).

Application Layer

Orchestration and services. EventBus handles pub/sub messaging. RiskService validates orders. ExecutionService routes to brokers. PositionTracker maintains state. KillSwitch provides emergency stop.

Ports Layer

Abstract interfaces (Python Protocols) that define contracts. BrokerPort, StreamPort, StorePort, ResearchPort, ClockPort. Domain depends only on these abstractions—never concrete implementations.

Adapters Layer

Concrete implementations that plug into ports. AlpacaBroker, AlpacaStream, TimescaleStore, ChromaOllamaResearch, LiveClock. Add new adapters without touching domain or application code.

Sports Analytics & Forecasting

Same architectural rigor applied to sports modeling. The platform's event-driven core and pluggable adapters translate directly to game prediction, season projections, and edge detection.

Matchup Simulations

Monte Carlo simulations for game outcomes. Domain models encode team strength, matchup factors, and situational adjustments. Run thousands of scenarios without touching data ingestion code.

Season Projections

Forward-looking season models that update as new data arrives. Event-driven architecture means projections recalculate automatically when game results stream in.

Custom Metrics

Build proprietary stats in the domain layer. Possession efficiency, pace-adjusted ratings, player impact scores—all computed from raw data adapters without coupling to specific APIs.

Pluggable Data Sources

Swap between sports data providers by implementing new adapters. Same port interfaces, same domain logic, different external feeds. No vendor lock-in.

Core Capabilities

Trading & Execution

  • Real-time WebSocket market data streaming
  • Market, Limit, and Stop order types
  • Strategy partitioning for multi-symbol trading
  • Graceful shutdown with order cleanup

Risk Management

  • Position size limits and concentration checks
  • Drawdown thresholds with automatic flattening
  • Circuit breakers for volatility spikes
  • KillSwitch for emergency system halt

Data Infrastructure

  • TimescaleDB for tick-level time-series storage
  • ChromaDB for vector embeddings and semantic search
  • DuckDB for analytics and RAG query logging
  • Event replay for backtesting and debugging

Research & RAG

  • PDF ingestion with semantic chunking
  • Natural language queries over research corpus
  • Local LLM generation via Ollama
  • Query rating and feedback loop

Quick Start

Get the trading engine running locally in five steps. Requires Python 3.11+, Docker, and Alpaca API credentials.

1

Clone & Setup

git clone https://github.com/kitchen-sys/kyzlo-quant.git
cd kyzlo-quant
python -m venv venv
source venv/bin/activate
2

Install Dependencies

pip install -r requirements.txt
3

Configure Environment

cp .env.example .env
# Edit .env with your API keys:
# ALPACA_API_KEY, ALPACA_SECRET_KEY
# TIMESCALE_CONNECTION_STRING
# OLLAMA_HOST (for RAG features)
4

Launch Infrastructure

docker-compose up -d
5

Run Trading Engine

python main.py
